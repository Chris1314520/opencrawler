认证方式

所有 API 请求需要携带 X-API-Key header，或在 URL 中传 ?api_key= 参数。

curl # Header 方式（推荐） curl -H "X-API-Key: sk-pro_YOUR_KEY" \ "http://your-server:5889/api/v1/trending?limit=10" # URL 参数方式（方便浏览器测试） curl "http://your-server:5889/api/v1/trending?api_key=sk-pro_YOUR_KEY&limit=10"

没有 Key 或 Key 无效会返回 401。超配额返回 429。

GET /api/v1/trending

获取聚合情报数据。支持按数据源、时间范围、关键词筛选。

参数

参数类型默认说明
sourcestringall数据源: github_trending, hackernews, rss, nvd_cve
limitint10返回条数（最大 50）
keywordstring-关键词筛选（标题+描述）
hoursint-最近 N 小时的数据

示例请求

curl curl -H "X-API-Key: sk-pro_xxx" \ "http://localhost:5889/api/v1/trending?source=nvd_cve&limit=5"

示例响应

JSON { "items": [ { "title": "CVE-2026-6419: WordPress plugin privilege escalation", "url": "https://nvd.nist.gov/vuln/detail/CVE-2026-6419", "description": "...vulnerability details...", "source_label": "CVE 漏洞", "tags": "安全,CVE,高危", "extra": { "cvss": "8.8 (HIGH)", "cve_id": "CVE-2026-6419" }, "fetched_at": 1780146731 } ], "total": 5, "tier": "pro", "quota": { "used_today": 3, "daily_limit": 500 } }

Python 示例

Python import requests resp = requests.get( "http://localhost:5889/api/v1/trending", headers={"X-API-Key": "sk-pro_xxx"}, params={"source": "nvd_cve", "limit": 10} ) data = resp.json() for item in data["items"]: print(item["title"])

GET /api/v1/search

关键词全文搜索，在标题和描述中匹配。

参数

参数类型默认说明
qstring必填搜索关键词
limitint20返回条数（最大 50）
curl curl -H "X-API-Key: sk-pro_xxx" \ "http://localhost:5889/api/v1/search?q=AI&limit=10"

限流说明

套餐日配额每分钟数据延迟
Free20 次5 次≤ 6 小时
Pro500 次30 次实时
Team不限60 次实时

超配额返回 429 Too Many Requests，配额每天 0:00 重置。如需提额请联系。

错误码

状态码含义返回内容
200成功正常数据
400参数错误{"error": "missing q parameter"}
401无 Key 或 Key 无效{"error": "missing api key"}
429配额用尽{"error": "daily quota exceeded", "limit": 20}
