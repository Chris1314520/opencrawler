前置环节：理解 OpenClaw 架构

OpenClaw 采用 Agent → Tool → LLM 三层架构。Agent 负责任务规划和决策，Tool 层提供浏览器/文件/命令行等工具，LLM 层对接大模型API。理解这个架构有助于你后续排错和自定义扩展。

  1. 克隆仓库并进入目录

    OpenClaw 托管在 GitHub，直接克隆即可。如果国内网络慢，可以使用 GitHub 镜像加速。

    bash 
    git clone https://github.com/browser-use/openclaw.git
cd openclaw
    国内网络加速：将 github.com 替换为 hub.fastgit.xyz 或使用 ghproxy.com 代理。
    git clone https://ghproxy.com/https://github.com/browser-use/openclaw.git

  2. 安装 Python 依赖

    OpenClaw 使用 Poetry 管理依赖。如果没有 Poetry，先安装它。

    bash 
    pip install poetry
poetry install
# 或使用 pip（如果项目有 requirements.txt）
pip install -r requirements.txt

  3. 配置 LLM API Key

    创建 .env 文件，填入你的大模型 API 密钥。支持多种模型同时配置，Agent 可根据任务自动选择。

    .env 
    # DeepSeek（推荐，性价比高）
DEEPSEEK_API_KEY=sk-your-deepseek-key
DEEPSEEK_BASE_URL=https://api.deepseek.com

# 通义千问（国内首选，延迟低）
QWEN_API_KEY=sk-your-qwen-key
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# Claude（能力最强）
ANTHROPIC_API_KEY=sk-ant-your-claude-key

# 默认使用的模型
DEFAULT_LLM_MODEL=deepseek-chat
DEFAULT_LLM_BASE_URL=https://api.deepseek.com
    注意：.env 文件包含敏感信息，不要提交到 Git。确保 .gitignore 中已包含 .env。

  4. 安装 Playwright 浏览器

    OpenClaw 使用 Playwright 驱动浏览器。需要安装 Chromium 内核。

    bash 
    playwright install chromium
# 如果下载慢，设置镜像
PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright playwright install chromium

  5. 启动 OpenClaw 并测试

    启动 Agent 服务，运行第一个任务来验证部署。

    bash 
    # 启动 Agent 服务（CLI 模式）
python main.py

# 或者用 Docker 启动
docker compose up -d

# 测试：让 Agent 搜索 GitHub trending
python main.py --task "搜索 GitHub 上今天最热门的 Python 项目"

  6. 配置开机自启（可选）

    生产环境下使用 systemd 确保 Agent 服务始终运行。

    ini 
    [Unit]
Description=OpenClaw Agent Service
After=network.target docker.service

[Service]
Type=simple
WorkingDirectory=/opt/openclaw
ExecStart=/usr/bin/docker compose up
ExecStop=/usr/bin/docker compose down
Restart=always

[Install]
WantedBy=multi-user.target

前置环节：Hermes 与 OpenClaw 的区别

Hermes (browser-use) 是一个纯粹的浏览器Agent框架，专注于网页操作。与 OpenClaw 不同，它不包含文件管理和命令行工具，但浏览器操控能力更强——支持视觉定位、拖拽、验证码识别。如果你的场景主要是网页数据采集、表单填写、网页监控，Hermes 更适合。

  1. 安装 browser-use

    Hermes 的核心是 browser-use 库。直接通过 pip 安装即可。

    bash 
    pip install browser-use
# 安装 Playwright 浏览器
playwright install chromium

  2. 创建第一个 Agent 任务

    编写 Python 脚本，让 Agent 自动完成网页操作。

    python 
    import asyncio
from browser_use import Agent
from langchain_openai import ChatOpenAI

async def main():
    # 使用 DeepSeek 作为 LLM（国内推荐）
    llm = ChatOpenAI(
        model="deepseek-chat",
        api_key="sk-your-key",
        base_url="https://api.deepseek.com",
    )

    agent = Agent(
        task="打开 GitHub Trending 页面，列出前10个最热门的仓库名称",
        llm=llm,
    )
    result = await agent.run()
    print(result)

asyncio.run(main())
    模型选择建议：浏览器Agent需要较强的视觉理解能力。推荐使用 Claude 3.5 Sonnet 或 Qwen-VL-Max 以获得更好的页面理解效果。DeepSeek 在文本理解上很强，但视觉能力较弱。

  3. 配置持久化浏览器上下文

    让 Agent 记住登录状态、Cookie等信息，避免每次重新登录。

    python 
    from browser_use import Agent, Browser, BrowserConfig

# 使用持久化用户目录
browser = Browser(
    config=BrowserConfig(
        chrome_instance_path="/usr/bin/google-chrome",
        user_data_dir="./browser_data",  # 保存登录状态
    )
)

agent = Agent(task="...", llm=llm, browser=browser)

  4. 部署为 API 服务

    将 Agent 封装为 REST API，方便其他服务调用。

    python 
    from fastapi import FastAPI
from pydantic import BaseModel
from browser_use import Agent

app = FastAPI()

class TaskRequest(BaseModel):
    task: str

@app.post("/agent/run")
async def run_agent(req: TaskRequest):
    agent = Agent(task=req.task, llm=llm)
    result = await agent.run()
    return {"result": str(result)}

# uvicorn server:app --host 0.0.0.0 --port 8000

OpenHuman 部署

多模态 Agent 框架，支持桌面操作、图像分析、视频理解。适合需要 GUI 自动化和视觉任务的场景。

前置环节：OpenHuman 的能力边界

OpenHuman 的独特之处在于桌面GUI操作 + 多模态理解。它能看到屏幕截图、理解窗口布局、模拟鼠标键盘操作。适合自动化测试、RPA（机器人流程自动化）、辅助办公等场景。但它对系统权限要求较高，建议在虚拟机或容器中运行。

  1. 安装 OpenHuman

    bash 
    git clone https://github.com/openhuman/openhuman.git
cd openhuman
pip install -e .
# 安装系统依赖
sudo apt-get install -y xvfb xdotool scrot python3-tk
    注意：OpenHuman 需要图形环境。在无头服务器上运行需要 Xvfb（虚拟显示）。macOS 用户需要额外的权限设置。

  2. 配置虚拟显示（服务器用）

    bash 
    # 启动虚拟显示
Xvfb :99 -screen 0 1920x1080x24 &
export DISPLAY=:99

# 验证
xdpyinfo -display :99 | grep dimensions
# 输出: dimensions: 1920x1080 pixels

  3. Docker 部署（推荐）

    使用 Docker 可以避免系统环境差异，一键启动完整的 OpenHuman 环境。

    dockerfile 
    FROM python:3.11-slim

RUN apt-get update && apt-get install -y \
    xvfb xdotool scrot python3-tk \
    wget curl git \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY . .
RUN pip install -e .

COPY entrypoint.sh /
RUN chmod +x /entrypoint.sh

ENV DISPLAY=:99
CMD ["/entrypoint.sh"]
    bash (entrypoint.sh) 
    #!/bin/bash
Xvfb :99 -screen 0 1920x1080x24 &
sleep 2
exec python -m openhuman "$@"

典型场景：每天自动监控竞品网站 → Hermes 采集页面变化 → OpenClaw 分析变化内容并生成摘要 → OpenHuman 打开 WPS 生成日报 → 推送到企业微信群。

实现方式：使用 n8n / Dify 工作流串联，或自己编写 Python 调度脚本。每个 Agent 独立部署在不同的容器中，通过 HTTP API 或消息队列（Redis/RabbitMQ）通信。

国内网络优化方案

解决 GitHub 访问慢、PyPI 下载失败、HuggingFace 无法连接等问题

问题解决方案配置方式
GitHub 克隆慢 使用 ghproxy.com 镜像 git clone https://ghproxy.com/https://github.com/xxx/xxx.git
PyPI 下载慢 清华/阿里云镜像 pip install -i https://mirrors.aliyun.com/pypi/simple/ xxx
Docker Hub 拉取慢 配置 Docker 镜像加速 /etc/docker/daemon.json: {"registry-mirrors":["https://xxx.mirror.aliyuncs.com"]}
HuggingFace 无法访问 hf-mirror.com 镜像 export HF_ENDPOINT=https://hf-mirror.com
Playwright 浏览器下载慢 npmmirror 镜像 PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright
npm 安装慢 npmmirror 镜像 npm config set registry https://registry.npmmirror.com
全局代理 HTTP/HTTPS 代理 export HTTP_PROXY=http://127.0.0.1:7890 HTTPS_PROXY=http://127.0.0.1:7890

常见问题排错

部署过程中最常见的问题和解决方案

Q: playwright 安装报错 "Browser closed unexpectedly"
A: 通常是缺少系统依赖。运行 playwright install-deps chromium 安装系统库。Docker 中需要以 root 运行或使用 --no-sandbox。
Q: Agent 执行任务时卡住不动
A: 检查 LLM API 是否可达（curl 测试），检查 API Key 是否有效，检查配额是否用完。国内访问 OpenAI 需要代理。
Q: Docker 容器内无法启动浏览器
A: 需要在 docker run 时添加 --cap-add=SYS_ADMIN 或使用 --security-opt seccomp=chrome.json。
Q: 中文网页乱码
A: 确保系统安装了中文字体：apt-get install fonts-noto-cjk，并设置 LANG=zh_CN.UTF-8。
